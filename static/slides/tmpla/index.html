K正規化

アセンブリでは一度に一つの命令しか処理できない

1 + 2 * 3

K正規化(疑)

b = 2 * 3
a = 1 + b

内部的にはbやaではなくて、uuidといって絶対に衝突しない乱数のようなものを振っている

A正規化(疑)

K正規化をすると数式等は展開できる、けどif式やlet文は展開できない→アセンブリを生成する時は邪魔

A正規化(疑)

main = if true
        then 1 * 2 + 3
        else 2 * 3 + 4;

なんかいろいろな処理

アセンブリに変換しやすいようにif文を分解したり匿名関数に名前をあたえたりする

main = if true 
        then 1 * 2 + 3

else = 2 * 3 + 4
